# Compose Screenshot Writer

## 역할

대상 화면으로 이동하는 **uiautomator2 navigation script**를 생성하고,
beta/production 앱에서 스크린샷을 캡처하여 작업 폴더에 저장한다.

---

## 입력

- `Screen`: 캡처 대상 Activity 클래스명 (예: `ListProfileActivity`)
- `Beta 패키지`: `com.dbs.kurly.m2.beta`
- `Production 패키지`: `com.dbs.kurly.m2` (before=production인 경우만)
- `작업 폴더`: `{WORK_DIR}`
- `캡처 모드`: `before` | `after`
- `스크립트 경로` (after 모드 전용): `{WORK_DIR}/capture_{SCREEN_NAME}.py`

---

## Step 1: 기기 연결 확인

```bash
adb devices | grep -v "List of devices" | grep "device$"
```

기기 없으면:
```
❌ ADB 기기가 연결되어 있지 않습니다. 기기를 연결하고 다시 시도하세요.
```
→ 즉시 중단.

## Step 2: [before 모드] 스크립트 탐색 또는 생성

### 2-1. 기존 스크립트 확인

`{WORK_DIR}/capture_{SCREEN_NAME}.py`가 이미 존재하면 Step 3으로 건너뜀.

### 2-2. 화면 진입 경로 파악

현재 feature 브랜치 코드에서 화면 진입 경로를 파악한다.

```bash
# Activity 클래스명으로 Intent/startActivity 호출처 탐색
grep -r "startActivity\|Intent(" --include="*.kt" -l | head -20
grep -r "{SCREEN_NAME}" --include="*.kt" -l | head -10

# Deep link 방식 확인 (NavDeepLink, @DeepLink 등)
grep -r "DeepLink\|deepLink\|navigation" --include="*.kt" -l | head -10
```

탐색 경로가 명확하지 않으면 **사용자에게 질문**:

```
❓ {SCREEN_NAME} 화면에 진입하는 경로를 알려주세요.
   예: "마이컬리 → 프로필 관리 → 목록형 프로필"
   또는 adb deep link가 있다면 URL도 알려주세요.
```

### 2-3. 스크립트 생성

uiautomator2 기반 Python 스크립트를 생성한다.
파일명: `{WORK_DIR}/capture_{SCREEN_NAME}.py`

```python
#!/usr/bin/env python3
"""
{SCREEN_NAME} Screenshot Capture Script
Generated by compose-screenshot-writer
"""
import uiautomator2 as u2
import time
import sys

PACKAGE = sys.argv[1] if len(sys.argv) > 1 else "com.dbs.kurly.m2.beta"
OUTPUT_PATH = sys.argv[2] if len(sys.argv) > 2 else "/tmp/screenshot.png"

d = u2.connect()

# 앱 시작
d.app_start(PACKAGE)
time.sleep(3)

# --- 화면 진입 경로 ---
# (파악된 진입 경로를 여기에 작성)
# 예시:
# d(resourceId=f"{PACKAGE}:id/tab_my_kurly").click()
# time.sleep(1)
# d(text="프로필 관리").click()
# time.sleep(1)
# d(text="목록형 프로필").click()
# time.sleep(2)

# 스크린샷 저장
d.screenshot(OUTPUT_PATH)
print(f"Screenshot saved: {OUTPUT_PATH}")
```

**진입 경로가 adb intent인 경우** 스크립트 대신 shell 명령 사용:
```python
import subprocess
subprocess.run([
    "adb", "shell", "am", "start",
    "-n", f"{PACKAGE}/.path.to.{SCREEN_NAME}",
])
```

스크립트 생성 후 사용자에게 확인 요청:
```
📝 capture_{SCREEN_NAME}.py를 생성했습니다.
   진입 경로를 확인해주세요. 수정이 필요하면 알려주세요.
   확인되면 'ok'를 입력해주세요.
```

→ 사용자 승인 후 Step 3 진행.

## Step 4: 스크린샷 캡처

### before 모드

**Beta 앱 캡처:**
```bash
python3 {WORK_DIR}/capture_{SCREEN_NAME}.py com.dbs.kurly.m2.beta /tmp/before_beta_tmp.png
adb pull /tmp/before_beta_tmp.png {WORK_DIR}/before_beta.png 2>/dev/null || \
  cp /tmp/before_beta_tmp.png {WORK_DIR}/before_beta.png
```

실제로 스크립트가 로컬에 PNG를 직접 저장하는 구조면 바로 확인:
```bash
ls -la {WORK_DIR}/before_beta.png
```

**Production 앱 캡처 (before=production인 경우):**
```bash
python3 {WORK_DIR}/capture_{SCREEN_NAME}.py com.dbs.kurly.m2 {WORK_DIR}/before_production.png
```

### after 모드

```bash
python3 {WORK_DIR}/capture_{SCREEN_NAME}.py com.dbs.kurly.m2.beta {WORK_DIR}/after_beta.png
```

## Step 5: 캡처 결과 확인

```bash
ls -la {WORK_DIR}/*.png
```

저장된 파일 목록 출력:
```
📸 캡처 완료:
  {WORK_DIR}/before_beta.png (before 모드)
  {WORK_DIR}/before_production.png (production 있는 경우)
  {WORK_DIR}/after_beta.png (after 모드)
```

파일이 없으면 에러 메시지와 함께 중단.

## 주의사항

- **스크립트 재사용**: after 모드에서는 기존 스크립트를 그대로 사용 (재생성 금지)
- **기기 없으면 즉시 중단**: ADB 없이 스크립트만 생성하지 않음
- **uiautomator2 element 탐색**: `d(text=...)` > `d(resourceId=...)` > `d(className=...)` 순서로 우선 사용
- **wait 전략**: `click()` 후 `time.sleep(1~2)` 충분히 부여하여 화면 전환 대기
- **스크롤 필요 시**: `d.swipe_ext("up")` 또는 `d(scrollable=True).scroll()` 사용
